¿Qué es un Servidor de aplicaciones?

¿Para qué se usa un servidor de aplicaciones?

  • Evitar las aplicaciones de escritorio
  • Aplicaciones "three-tier"(GUI,Lógica Empresarial,Base de Datos)
  • Soporte de sitio web

¿Por qué usar un servidor de aplicaciones?

  • Escalabilidad
  • Disminuye la complejidad del desarrollo
  • Buen rendimiento
  • Seguridad

GlassFish

Ventajas

  • Gratuito
  • Código abierto
  • Integración total con NetBeans
  • Linux y Windows
  • Consola de administración súper-chida

Desventajas

  • Gratuito(Mantenimiento)
  • Difícil integración con eclipse.
  • Sacrifica velocidad para buscar ser más robusto.
  • No desconecta las conexiones a la BD y luego se cae por muchas conexiones.
